Component Diagram

AnzaConnect System Architecture

Unidirectional data flow — screens dispatch events to BLoC state machines, which call repositories, which read from and write to Firebase backend services.

UI Flutter Screens Widgets, user interactions & navigation via go_router
FeedScreen Opportunity feed + bookmarks
OpportunityDetail Role, deadline, apply flow
ApplyScreen Cover letter + doc upload
BookmarkScreen StreamBuilder on saved items
StudentProfile Photo, name, applications
StartupDashboard Post opportunities, view apps
AdminDashboard Verify / revoke startups
LoginScreen Email & Google sign-in
STATE BLoC Layer Events in → state out. Screens listen via BlocBuilder
FeedBloc LoadFeed, BookmarkedIdsUpdated
FeedLoaded opportunities, bookmarkedIds, studentName
AuthBloc LoginRequested, RegisterRequested
AuthAuthenticated role-based navigation on emit
StartupBloc LoadStartup, PostOpportunity
StartupLoaded startup map + opportunity list
DATA Repository Layer Encapsulates Firestore queries and external service calls
OpportunityRepository getOpenOpportunities(), sort by date
BookmarkRepository addBookmark, watchBookmarks stream
ApplicationRepository submitApplication, updateStatus
StartupRepository getAllStartups, setVerified
AuthService signIn, signUp, signOut, isAdmin
R2StorageService AWS4-HMAC-SHA256 signed upload
BACKEND Firebase & Services Persistent storage, authentication & external integrations
